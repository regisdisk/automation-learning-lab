# Retrospective — Workflow 05

Date:
2026-05

Release:
v0.5.0

Workflow:
Event-Driven Action Orchestration

---

## Objective

Convert operational events into automated actions through a reusable orchestration pipeline.

---

## Architecture

Event ingestion gateway
-> n8n
-> Home Assistant action provider

---

## What Worked

- Webhook-based event ingestion
- Conditional routing in n8n
- Home Assistant provider state
- State cleanup automation
- End-to-end validation

---

## What Failed

- Gmail IMAP approach
- Alexa Media Player authentication
- Assuming HTTP 200 meant completed action

---

## Decisions Taken

Keep:

Event
-> Decision
-> State
-> Action

Avoid:

Event
-> Direct Device Action

---

## Mental Models Reinforced

Producer ≠ Publisher

Decision ≠ Execution

HTTP Accepted ≠ Outcome Confirmed

State ≠ Automation

---

## Technical Debt

- Alexa integration incomplete
- No delivery confirmation from external notifications
- Authentication flow still manual

---

## Assets Generated

- Workflow
- Validation tests
- Release
- Troubleshooting guides

---

## What To Reuse

Webhook pattern

State-based automation

Decision separation
