# Roadmap

This roadmap organizes the lab as a progression from automation foundations to observability, operational intelligence and workflow governance.

## Target Sequence

| Workflow | Focus | Status |
|----------|-------|--------|
| 01 - Health Check | Runtime validation and workflow availability | Completed |
| 02 - Webhook Echo | HTTP ingress and request/response behavior | Completed |
| 03 - Site Monitor | External service monitoring and alert conditions | Completed |
| 04 - Event Ingestion Gateway | Trusted event ingestion, transformation and normalization | Completed |
| 05 - Event-Driven Action Orchestration | Event-to-action routing through external action providers | Completed |
| 06 - Stateful Alert Management | Deduplication, cooldowns and alert lifecycle control | Planned |
| 07 - Workflow Persistence Layer | Historical execution data and queryable operational state | Planned |
| 08 - Operational Dashboard Feed | Dashboard-ready telemetry and observability outputs | Planned |
| 09 - Voice Notification Layer | Channel-independent notification delivery | Discovery |

## Architectural Progression

Automation -> observability -> operational intelligence.

The early workflows establish execution and integration primitives. The middle workflows introduce event ingestion, normalization and orchestration. The later workflows add state, persistence, dashboard feeds and governed notification channels.
